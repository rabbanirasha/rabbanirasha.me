Description of the Project

This program generates PDF of bill and renames its filename for each DP automatically by varying the DPID, bill_month and bill_date string of a web link in a loop, from a recipient list of DP IDs (usually about 400 entries) in an excel file. This eased off the workload of assigned person as only bill date and period is entered once and the rest is done by the program, unlike previously where the person had to repeat the process of entering ID of a DP (Depository Participant) and bill info to Oracle based software and then renaming the generated file hundred of times for all DPs.

The sequence is implemented using UIPath Studio and the custom input page is made using HTML and Javascript.

My Role

I solely developed this program for CDBL after finding out that generating bills required repetitive manual input and manipulation.

Description of the Project

A web-based responsive and dynamic software built from a custom framework, that accounts valuation and details of every asset of the company, and also provides insights of these assets. Assets include any possession of CDBL including anything from chairs, tables, PCs, electronics, software to lands and automobiles. Notable features include:

  • Multi-user platform with complete account management via user-based and role-based access control with ability of superadmin to grant users different priviledges to view and edit data by creating roles and users with necessary priviledges. This results in personalized homepage of that user.
  • Detailed analytics of assets based on category and departments under which it is deployed.
  • Backend database not needed to be touched because all actions like adding, modifying, deleting users, departments, assets and categories can be done within the software.
  • Licenses can be assigned to assets or vice-versa. Assignment can be done both ways - one license can be assigned to many assets or each asset can be given corresponding license.
  • A notification system, via bell icon on header and Email/SMS alerts, to let respective users know what assets have expired, or had its location or corresponding department changed.
  • Timesheet report (summary of all assets and licenses in the selected time period), department report(summary of all assets and licenses under the department),user report (summary of all assets assigned to a user),other reports (department/location change) generation.
  • Maintenance log for each asset.
  • Complete system log with IP address and timestamp for every change occuring in its database, like adding new user, who logged in, asset edit, etc.

Technology

PHP, MySQL, HTML, CSS, MD5 and SHA1 hash, Javascript (mainly JS Plugins)

My Role

I developed the notification system of its dashboard and fixed bugs and also worked on frontend and backend data fields of each page whenever necessary.

Description of the Project

A new official website of CDBL (www.cdbl.com.bd/beta) built from the ground up for the modern web and devices which is dynamic and responsive with real-time data presentation and less amount of navigation, while keeping the actions as consistent as before so visitors do not have to face steep learning curves.
